# 图与景区信息管理系统

基于图算法实现的景区信息管理系统，提供景点导航、路径规划、电路铺设等功能。

## 实验目的

通过"景区信息管理系统"的编程实践，学习图的遍历、Dijkstra算法、最小生成树算法、Prim算法和它们的编程应用。

- 掌握图的定义和图的存储结构
- 掌握图的创建方法
- 掌握图的两种遍历方法
- 理解迪杰斯特拉（Dijkstra）算法
- 理解最小生成树的概念和普里姆（Prim）算法
- 掌握文件操作
- 使用C++语言，利用图的数据结构，开发景区信息管理系统

## 业务背景

现有一个景区，景区里面有若干个景点，其景点分布如图1所示，并且景点之间满足以下条件：

1. 某些景点之间铺设了道路（相邻）
2. 这些道路都可以双向行驶的（无向图）
3. 从任意一个景点出发都可以游览整个景区（连通图）

开发景区信息管理系统，对景区的信息进行管理。使用图的数据结构来保存景区景点信息，为用户提供创建图、查询景点信息、旅游景点导航、搜索最短路径、铺设电路规划等功能。

## 景点数据

景区的数据包含景点信息和景点之间的道路信息，分别由两个文本文件存储。

### 景点信息（Vex.txt）

景点信息主要包含景点的编号、景点的名字和相关介绍等内容：

| 编号 | 名字 | 介绍 |
|------|------|------|
| 0 | A区 | A区是景区的主要入口，有停车场和游客中心 |
| 1 | B区 | B区有儿童游乐场和休息区 |
| 2 | C区 | C区有历史博物馆和文化展览 |
| 3 | D区 | D区有自然风光和观景台 |
| 4 | E区 | E区有植物园和花卉展览 |
| 5 | F区 | F区有水上乐园和沙滩 |
| 6 | G区 | G区有山顶观景台和索道 |

### 道路信息（Edge.txt）

道路信息通过一个三元组的形式来表示，分别是景点1（起点）的名字，景点2（终点）的名字，两个景点之间的距离长度：

| 景点1 | 景点2 | 距离（m） |

## 系统功能

整个景区信息管理包含5个功能模块，分别是创建图，查询景点，旅游景点导航，搜索最短路径和铺设电路规划。

### 1. 创建图

**输入：** 从Vex.txt文件中读取景点信息，从Edge.txt文件中读取道路信息。

**处理：** 根据读取的景点信息创建景区景点图。

**输出：** 创建成功，一次输出：
1. 顶点数目
2. 顶点编号
3. 顶点名字
4. 边两端的顶点编号
5. 边的权值

创建失败，输出失败的提示信息。

### 2. 查询景点

**输入：** 想要查询的景点的编号

**处理：** 根据输入的景点编号，查询该景点及相邻景点的信息。

**输出：**
1. 景点名字
2. 景点介绍
3. 相邻景点的名字
4. 到达相邻景区的路径长度

### 3. 旅游景点导航

**输入：** 起始景点的编号

**处理：** 使用深度优先搜索（DFS）算法，查找以该景点为起点，无回路游览整个景区所有景点的所有可能路线（哈密顿路径）。优先选择距离最短的边进行探索。

**输出：**
1. 所有符合要求的导航路线
2. 每条路线的总距离

### 4. 搜索最短路径

**输入：** 起始景点的编号和终点的编号

**处理：** 使用迪杰斯特拉算法，求得从起始景点到终点之间的最短路径，计算路径总长度。

**输出：**
1. 最短路线
2. 路径总长度

### 5. 铺设电路规划

**输入：** 景区的所有景点信息和道路信息

**处理：** 根据景区景点图使用普里姆（Prim）算法构造最小生成树，设计出一套铺设线路最短，但能满足每个景点都能通电的方案。

**输出：**
1. 需要铺设电路的道路
2. 每条道路铺设电路的长度
3. 铺设电路的总长度

## 核心算法

### DFS（深度优先搜索）
用于旅游景点导航，查找无回路游览整个景区所有景点的路线（哈密顿路径）。

**算法思路：**
1. 从起始顶点开始，标记为已访问
2. 递归访问所有未访问的邻接顶点（优先选择最短边）
3. 当路径长度等于景点总数时，保存该路径
4. 回溯继续探索其他分支

**特点：** 返回所有能遍历全部景点的无回路路线

**时间复杂度：** O(V!)，最坏情况需要枚举所有排列

### Dijkstra算法
用于搜索最短路径，计算两点间的最短路径和距离。

**算法思路：**
1. 初始化距离数组，起点距离为0，其余为无穷大
2. 每次选择未访问顶点中距离最小的顶点
3. 对该顶点的邻接顶点进行松弛操作
4. 重复直到所有顶点都被访问

**时间复杂度：** O(V²)，可优化至O(E log V)（使用优先队列）

### Prim算法
用于铺设电路规划，构造最小生成树。

**算法思路：**
1. 从任意顶点开始，将其加入最小生成树
2. 在所有连接已访问和未访问顶点的边中选择权值最小的边
3. 将该边对应的未访问顶点加入最小生成树
4. 重复直到所有顶点都被访问

**时间复杂度：** O(V²)，可优化至O(E log V)（使用优先队列）

## 算法复杂度分析

| 算法 | 时间复杂度 | 空间复杂度 | 适用场景 |
|------|-----------|-----------|---------|
| DFS（哈密顿路径） | O(V!) | O(V) | 遍历所有景点的无回路路线 |
| Dijkstra | O(V²) | O(V) | 单源最短路径 |
| Prim | O(V²) | O(V) | 最小生成树 |

## 数据文件格式

### Vex.txt 格式
```
<编号> <名称> <描述>
```
- 编号：整数，从0开始
- 名称：字符串，不含空格
- 描述：字符串，可包含空格

示例：
```
0 A区 A区是景区的主要入口，有停车场和游客中心
```

### Edge.txt 格式
```
<起点名称> <终点名称> <权重>
```
- 起点名称：字符串，需与Vex.txt中的名称一致
- 终点名称：字符串，需与Vex.txt中的名称一致
- 权重：整数，表示距离（单位：米）

示例：
```
A区 C区 700
```
